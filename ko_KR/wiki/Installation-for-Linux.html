Jamulus 사용 | 시작하기

Linux용 설치

시작하기 페이지를 반드시 읽으십시오.

Jamulus 3.9.1은 이전 페이더 레벨이 손실될 수 있는 MIDI 컨트롤러 관련 버그를 해결합니다. `--ctrlmidich`를 사용한 경우 `--cleanuplegacyfadersettings`로 앱을 한 번 시작하여 이전 형식을 변환할 수 있습니다. 이 옵션은 이후 릴리스에서 제거될 예정입니다. 자세한 내용은 변경 로그를 참조하십시오.

업그레이드중입니까? 먼저 사용자의 구성 백업을 원할 수 있습니다.

데비안 및 우분투

가장 일반적인 아키텍처에 대해 3개의 동등한 .deb 파일을 제공합니다. 적절한 것을 다운로드하십시오:

인텔/AMD 기반 시스템의 경우 (amd64):

Jamulus (.deb, amd64) 다운로드

ARM 기반 시스템의 경우 (예: 라즈베리 파이, armhf, arm64):

Jamulus 32비트(.deb, armhf) 다운로드Jamulus 64비트 (.deb, arm64) 다운로드

올바른 파일을 다운로드한 후:

    • 우분투 전용 * - 우분투 “universe”저장소를 활성화합니다(GUI 기반 접근 방식 또는 CLI 기반 접근 방식).
  2. 콘솔 창을 열어 apt를 업데이트하고(CTRL+ALT+T가 작동해야 함) sudo apt-get update를 입력합니다
  3. 설치 프로그램을 다운로드 한 위치로 이동하여 두 번 클릭하거나, 명령 줄을 사용합니다: sudo apt install ./jamulus_3.9.1_ubuntu_amd64.deb armhf의 경우: sudo apt install ./jamulus_3.9.1_ubuntu_armhf.deb arm64의 경우: sudo apt install ./jamulus_3.9.1_ubuntu_arm64.deb
  4. 설치가 완료되면 파일을 삭제하고 콘솔 창을 닫을 수 있습니다.

Jamulus를 최신 버전으로 업그레이드해야 하는 경우, 새 .deb 파일을 다운로드하고 위와 같이 다시 설치하면 됩니다.

다른 배포판

다른 배포판의 설치 프로그램은 패키지 관리자 및 Repology를 참조하십시오. 최신 버전의 Jamulus가 배포판에 포함되어 있지 않은 경우 소스에서 Jamulus 컴파일을 할 수 있습니다. 제공된 설치 스크립트도 참고하십시오.

사용자의 하드웨어 설정

QjackCtl로 JACK 구성

Jamulus 클라이언트를 실행하려면 JACK가 필요하지만, 먼저 구성해야 합니다. 권장되는 방법은 QjackCtl을 사용하는 것입니다.

  1. QjackCtl을 실행합니다. Qt JACK 컨트롤 유틸리티 기본 페이지가 표시됩니다
  2. 오디오 하드웨어를 다음과 같이 구성합니다(JACK에 대한 정확한 설정은 오디오 하드웨어의 기능에 따라 다름):
  • 오디오 인터페이스를 원하는 것으로 설정합니다(목록에 여러 개 있을 수 있음)
  • 샘플 속도를 48000으로 설정합니다
  • 프레임/주기 128로 설정하고 주기/버퍼를 처음에는 2로 설정합니다

새 설정을 적용하려면 JACK을 다시 시작합니다

Jamulus 시작하기

JACK이 실행되고 구성된 상태에서 Jamulus를 시작합니다.

사운드 끊김 문제(특히 JACK/QjackCtl에 의해 보고된 XRUN 오류)가 발생하면 더 큰 값(예: 256프레임 또는 3주기)을 설정해 보십시오. 낮은 프레임(예: 64 프레임)은 더 나은 성능을 제공할 수 있지만, 더 많은 사운드 문제를 가져올 수 있습니다. 그렇지 않으면 문제 해결 페이지를 참조하십시오.
